# AGENTS.md

このファイルは、このリポジトリで作業する Codex 向けの運用メモです。  
ユーザー向け説明は `README.md`、作業方針と編集時の注意はこのファイルにまとめます。

## プロジェクト概要

- 対象はブラウザのみで動くバスチャイム生成ツール
- 実装は `index.html` 単体で完結する
- 外部ライブラリ、CDN、サーバ利用は禁止
- 技術要素は HTML / CSS / JavaScript / Web Audio API / OfflineAudioContext のみ

## 最優先事項

1. UI より音質を優先する
2. 単純なビープ音に戻さない
3. 公共交通の案内音としての上品さを保つ
4. 既存機能を壊さない

## 現在の主な機能

- 3点チャイム / 4点チャイム切り替え
- 3種類の音色プリセット
  - `Standard Bell`
  - `Soft Chime`
  - `Bright Synth Chime`
- 音の高さ、テンポ、音量の調整
- 即時試聴
- WAV ダウンロード

## コードの見方

`index.html` は大きく次のブロックで構成される。

- CSS
  - 画面全体のレイアウトと操作UI
- `PRESETS`
  - 音色ごとのパラメータ定義
- `PATTERNS`
  - 3点 / 4点チャイムの音程比
- UI 関連関数
  - `getSettings()`
  - `updateUi()`
- 音声初期化
  - `ensureAudioContext()`
- 音作り関連
  - `createDriveCurve()`
  - `createMasterChain()`
  - `createReverbNetwork()`
  - `createAttackNoiseBuffer()`
  - `scheduleTransient()`
  - `scheduleVoice()`
  - `renderChimeToContext()`
- 書き出し関連
  - `audioBufferToWavBlob()`
  - `downloadWav()`

## 作業時の基本方針

- 変更前に小さい計画を共有する
- 完了後は変更概要を説明する
- 大きく構造を変える前に、既存機能への影響を意識する
- UI の見た目変更より、音源ロジック改善を優先する
- 再生処理と書き出し処理は、できるだけ同じ音源ロジックを共有する

## 標準の進め方

以下を毎回の標準フローとする。

1. 作業前に小さい計画を共有する
2. 実装を進める
3. 完了後に、何をしたかを短く要約して説明する

## 音作りで守ること

- 基音だけで終わらせず、倍音を含める
- 厚みを出すためのデチューンは控えめにし、不協和にならない範囲に収める
- アタックは明瞭にするが、刺さる高音にはしない
- 減衰は急に切らず、自然に落とす
- クリックノイズ防止のため、ゲインや周波数の変化は必ず滑らかにする
- 残響は過剰にせず、案内音としての明瞭さを残す
- 音量を上げすぎず、必要なら最終段で軽く整える

## 変更時のチェックポイント

- プリセット変更直後でも再生できるか
- スマホでユーザー操作後に再生できるか
- WAV 書き出しが正常に動くか
- 試聴音と書き出し音でキャラクターが大きくずれないか
- 音が歪みすぎていないか
- ノートの頭や終端でクリックノイズが出ていないか
- 3つのプリセット差が明確に感じられるか

## よくある改善候補

- リバーブや反射音の設計改善
- マスター段の音量バランス調整
