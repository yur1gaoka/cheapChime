# project-profile.md

## 目的
本プロジェクトの確定条件を定義する
未確定事項を勝手に補完せず、確認済み条件と保留条件を分けて管理する

---

## 技術スタック
- HTML
- CSS
- JavaScript
- Web Audio API
- OfflineAudioContext

## 実行環境
- ブラウザで `index.html` を直接開いて実行する単一ファイル構成
- PCブラウザでの再生を想定
- スマホではユーザー操作後の音声有効化を前提に再生

## Docker
- 該当なし
- 現行実装はブラウザのみで完結し、リポジトリ内にも Docker 関連設定は確認していない

## DB
- 該当なし
- 現行実装はクライアント単体で完結し、DB 利用箇所は確認していない

## 本番想定
- ブラウザ上で案内音向けチャイムを生成し、その場で試聴または WAV 書き出しして利用する
- 配布単位は `index.html` 単体

## 制約
- `index.html` 単体で完結すること
- 外部ライブラリ、CDN、サーバを追加しないこと
- 音質を UI より優先し、単純なビープ音へ戻さないこと
- 公共交通の案内音としての上品さを保つこと
- 再生処理と書き出し処理はできるだけ同じ音源ロジックを共有すること

## 記入ルール
- 確認済みの事実のみ記入する
- 不明な場合は「未確認」、確認待ちは「確認待ち」、適用外は「該当なし」と明記する
- 推測で「利用なし」「不要」などの断定を書かない
- 後続判断に影響する項目は空欄で残さない
- 更新時は関連文書との整合を確認する

---

## ルール

- 不明な項目がある場合は実装禁止
- 更新時は一貫性を保つ
- 本ファイルの記入内容を着手判断の基準にする
